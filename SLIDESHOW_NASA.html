The Problem: No Autonomous Cross-Operator Authorization

  • No standard for cross-operator satellite authorization
  • Inter-satellite tasking requires bilateral legal agreements
  • Emergency response waits hours for ground station passes
  • No mechanism for trustless payment between operators

The Vision: Autonomous Multi-Operator Coordination

Cryptographic Authorization + Trustless Settlement

  • SISL: Secure Inter-Satellite Link - mutual authentication without pre-shared secrets
  • SCAP: Capability tokens authorize tasks, atomically bound to payment
  • Result: Any satellite can request services from any operator, settled trustlessly

SISL: Secure Inter-Satellite Link Protocol

Mutual Authentication Without Pre-Shared Secrets

Layer Function
SCAP Application Capability tokens, task requests, payments
SISL Security Sublayer X3DH key agreement, AES-256-GCM encryption
Data Services Fragmentation, selective ARQ
Physical Layer DSSS/FHSS spread spectrum (LPI/LPD)
  • X3DH mutual authentication: No signatures over nonces (eliminates nonce reuse catastrophe)
  • Forward secrecy: Ephemeral keys deleted after session
  • No pre-shared secrets: Works between satellites that have never communicated
  • CCSDS compliant: AES-256-GCM per Space Data Link Security (SDLS)

SCAP: Satellite Capability and Payment Protocol

Authorization + Payment in One Cryptographic Package

CAPABILITY TOKEN (SAT-CAP)
Issuer: "ESA-COPERNICUS" (Target's operator signs)
Subject: "ICEYE-X14" (Who can command)
Audience: "SENTINEL-2C" (Target satellite)
Capabilities: ["cmd:imaging:msi", "cmd:attitude:point"]
Constraints: {max_range_km: 100, geographic_bounds: {...}}
Payment: {adaptor_point: "02abc...", amount_sats: 50000}
Expiration: 24-48 hours
  • Delegation chains: Authority can be delegated, but only narrowed
  • Offline verification: Satellites verify tokens without ground contact
  • Atomic settlement: Task + payment complete together via adaptor signatures

Capability Delegation: Authority Without Trust

Pre-Authorized Emergency Response

  • Pre-issued emergency tokens: Sub-minute response without ground contact
  • Wildcard subjects: "IRIDIUM-*" enables any Iridium satellite to re-delegate
  • Offline verification: Chain validated locally using operator public keys

Atomic Settlement: Adaptor Signatures

Task Completion Cryptographically Bound to Payment

PropertyGuarantee

FairnessRequester can't claim without receiving proof
Timeout6-hour window; defaults to executor (real costs incurred)

Result: Either task completes AND payment settles, OR timeout refunds requester

Protocol Architecture: End-to-End Security

Authorization in Space, Settlement on Ground

Key insight: Satellites authorize and execute tasks autonomously. Ground operators handle settlement when connectivity permits.

WHAT AN ATTACKER CANNOT DO:
  • Forge capability tokens
  • Replay tokens (unique jti + cache)
  • Steal payments (timeout refund)
  • Eavesdrop on ISL (AES-256-GCM)
  • Impersonate satellites (X3DH)
